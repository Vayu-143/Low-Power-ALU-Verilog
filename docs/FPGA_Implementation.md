# FPGA Implementation Guide

# Low-Power ALU Design Using Verilog

## Overview

This document describes the FPGA implementation flow for the Low-Power ALU Design project. The design is developed using SystemVerilog and can be synthesized and implemented on FPGA platforms such as the Xilinx Basys 3, Artix-7, Nexys A7, or similar development boards.

Even if FPGA hardware is unavailable, synthesis, implementation, and utilization reports generated by Vivado serve as strong proof of design correctness and FPGA readiness.

---

# FPGA Implementation Objectives

The objectives of FPGA implementation are:

* Verify synthesizability of the RTL design
* Generate FPGA resource utilization reports
* Analyze timing performance
* Analyze estimated power consumption
* Demonstrate hardware readiness
* Validate RTL design flow used in industry

---

# Target FPGA Platforms

The design can be implemented on:

* Xilinx Basys 3


# Example Hardware Mapping

## Inputs

```text
SW0-SW7     -> Operand A

SW8-SW15    -> Operand B

SW16-SW19   -> Opcode

BTN0        -> Enable
```

---

## Outputs

```text
LED0-LED7   -> Result

LED8        -> Zero Flag

LED9        -> Carry Flag

LED10       -> Overflow Flag

LED11       -> Negative Flag
```

Check:

```text

```

Important Metric:

```text

```

Expected:

```text
WNS >= 0
```

Positive slack indicates timing closure.

# Low-Power Verification

Operand isolation reduces unnecessary switching activity.

RTL implementation:

```systemverilog
A_iso = enable ? A : 0;
B_iso = enable ? B : 0;
```

Benefits:

* Reduced switching activity
* Lower dynamic power
* Improved energy efficiency
